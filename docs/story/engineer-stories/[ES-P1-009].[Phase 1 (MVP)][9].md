### ES-P1-008 - API Service BFF Orchestration Entrypoint for MVP Web Flows
- Task Description:
	- Implement `api-service` BFF orchestration entrypoint for MVP web flows across auth, jam, playback, and catalog.
	- Define request routing and response aggregation behavior so web client calls can use one BFF-facing API surface.
	- Standardize upstream error mapping and timeout handling for auth, jam, playback, and catalog dependencies.
- Dependencies:
	- Stable downstream contracts from `auth-service`, `jam-service`, `playback-service`, and `catalog-service`.
	- API gateway routing and identity propagation into `api-service`.
	- Shared response envelope and error contract alignment across services.
- Definition of Done:
	- `api-service` provides a working BFF entrypoint for MVP web client integration.
	- MVP web flows (auth, jam, playback, catalog) can be invoked through the BFF path with deterministic behavior.
	- Integration tests verify routing, aggregation, and failure mapping for core MVP paths.
